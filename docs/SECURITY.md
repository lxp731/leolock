## 🔐 安全特性详解

LeoLock 始终将数据安全性放在首位。以下是本项目核心安全机制的详细说明。

### 1. 密码学算法与实现
- **对称加密**: 使用 **AES-256-GCM** (V3 格式)。这是一种认证加密算法，不仅保证数据的私密性，还能检测任何对加密数据的非法篡改。
- **元数据保护 (AAD)**: 文件头元数据（如版本号、文件名长度）被作为 **附加认证数据 (AAD)** 参与加密计算。任何对文件头的修改都将导致解密失败。
- **密钥派生**: 使用 **Argon2id** 算法从用户密码派生 256 位主密钥。该算法具备极强的抗 GPU/ASIC 暴力破解能力。
- **流式处理**: 采用 **1MB 分块加密**。每个分块都有唯一的随机 Nonce，确保了即使在处理数 GB 大文件时，安全性也不会降低。

### 2. 内存安全 (Memory Hardening)
- **零残留策略**: 集成 `zeroize` 库。用户密码、派生密钥以及解密后的文件名敏感数据，在离开作用域时会通过 RAII 机制被**物理擦除 (Zeroed)**，防止内存嗅探攻击。
- **敏感数据包装**: 使用 `Zeroizing<T>` 智能指针包装敏感字符串和字节数组，确保其在堆内存释放前被清零。

### 3. 密码管理与策略
- **多样化来源**:
  - **交互式**: 默认安全输入（无回显）。
  - **系统钥匙串**: 支持通过 `--keyring` 从 OS 系统钥匙串（如 Linux Secret Service, macOS Keychain）自动加载。
  - **环境变量**: 支持通过 `--env-pass` 指定变量名加载。
  - **标准输入**: 支持通过 `--stdin` 从管道接收密码。
- **强度评估**: 实时检测密码复杂度，强制要求至少 8 位，并提供基于数字、大小写字母及符号的强度反馈。

### 4. 运行时安全防护
- **原子替换**: 写入操作先生成 `.tmp` 文件，校验成功后执行原子重命名。防止系统崩溃导致的文件损坏。
- **危险路径保护**: 默认禁止对 16 个核心系统目录（如 `/etc`, `/usr/bin`）进行加密操作，防止误操作导致系统崩溃。
- **文件权限**: 配置文件和密钥文件默认权限设为 `600`。如果权限过大，工具在运行时会发出警报。

### 5. 灾难恢复
- **加密备份**: `init` 时生成包含主密钥的加密备份文件（使用 Argon2id 加密）。
- **完全离线**: 所有操作均在本地完成，绝不上传任何密钥或数据到云端。

---

**安全建议**: 尽管 LeoLock 提供了多种加载密码的方式，但在多用户环境下，建议优先使用 `--keyring` 或交互式输入，避免将明文密码留在 shell 历史记录中。
